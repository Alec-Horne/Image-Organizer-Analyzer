Help:

The webpage requires a valid login to use. You may register if you need to at the home page.

After logging in you will be taken to your albums page. Here you can add or delete pictures to your available albums. There are 5 with 10 spaces each.

You can use custom filters/preset filters or analyze your pictures by using the "process" selection on the nav bar. Your albums will appear on the right column and you can drag and drop them onto the canvas.

The preset filter page gives you preset filters on buttons you can apply. After you are satisfied with your changes you can either save it or use a different picture.

The custom filters page allows you to adjust sliders of values such as contrast, brightness, saturation, etc. to fine tweak your pictures. You may also save the final result here as well.

The analyze page will provide you with a histogram of each color value (RGB) and the average color both visually and in hexadecimal form.

To log out of the system please click the home link on the top navigation bar and there will be a button to log out.

Security:

For security, each page requires the user to be logged in, or more specifically the session variable "username" must be set. This is set only when the user sucessfully logs in and the password and username are checked against the database. Then each page has a simple check at the top to see if this is true. If it is not, the user is redirected to the login/register page.

Database Design:

For the database we used a total of 6 different tables. The first one simply holds the user login, password, name, and e-mail address. The other 5 are for the picture albums of each user. Each album table has a place for the album name, and then there are 10 entries for each picture. The picture is not held in the database but rather the path name that leads to its location on the server. All pictures are uploaded to the uploads folder.

Architecture:

The architecture for this project is pretty simple. First off, each user must be logged in to get any further on the website. Each php page checks to see if the user is logged in before displaying its contents. If not they are redirected to the login/register page. After they login they are taken to their albums page. From here the user can add photos or delete them from their albums. The analyze and filter pages all process the album information from the database and allow the user to drag and drop them onto the canvas to be modified/analyzed.
